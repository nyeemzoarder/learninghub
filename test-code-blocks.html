Code Block Styling Test

Compare the old (darker) vs new (brighter) code block styling

PowerShell Code Example

❌ OLD (Dark - Hard to Read)

# Remove policy assignments
Remove-AzPolicyAssignment -Name "Require a tag on resource groups" -Scope "/subscriptions/[subscription-id]"
Remove-AzPolicyAssignment -Name "Allowed Azure Locations" -Scope "/subscriptions/[subscription-id]"

# Remove custom role
Remove-AzRoleDefinition -Id "[role-id]" -Force

# Remove resource group
Remove-AzResourceGroup -Name "rg-az104-lab02" -Force

✅ NEW (Bright - Easy to Read)

# Remove policy assignments
Remove-AzPolicyAssignment -Name "Require a tag on resource groups" -Scope "/subscriptions/[subscription-id]"
Remove-AzPolicyAssignment -Name "Allowed Azure Locations" -Scope "/subscriptions/[subscription-id]"

# Remove custom role
Remove-AzRoleDefinition -Id "[role-id]" -Force

# Remove resource group
Remove-AzResourceGroup -Name "rg-az104-lab02" -Force

  • Lighter background (#2d2d30 vs #1e1e1e)
  • Better text contrast (more readable)
  • Colored syntax highlighting:

    • 🔵 Blue properties
    • 🟠 Orange strings
  • Better readability for long code blocks
